Security of HILS.

HILS is for at home only

HILS is intended to be used at home, on private networks. It is NOT intended to be exposed to the internet. This is because there are some security compromises to aid development and make it as user friendly as possible.

Please do not expose HILS to the internet. It is NOT designed to be and may expose your network to adversaries.

Why are there compromises?

I am not a professional developer and do not intend to become one. HILS is a hobby project and as a result, I don't want to spend my time patching up endless security exploits. I've taken some basic, reasonable, steps to keep this secure, but you are on your own and I offer no guarantees of security.

The SQLite3 Database File

The HILS database is a SQLite3 database, stored on your local server. It is not encyrpted in any way.

Anyone can access it and change it if they can access the file or your server.

You, and you alone, are responsible for keeping it safe.

Password Hasing

HILS will hash passwords according to whatever the PHP default is. Unless you have changed something, it is probably bcrypt. No salting or any other fancy things are done to the passwords in the database. This is because HILS is intended for internal, at home use and is not intended to be exposed to the internet.

Thank you for reading this notice.
